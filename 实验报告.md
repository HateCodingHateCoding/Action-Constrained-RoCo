# 智能系统期末项目 · 实验报告

## 基于强化学习的多机械臂协作动作约束方法

---

## 1. 项目概述

### 1.1 研究背景

RoCo（Dialectic Multi-Robot Collaboration）框架利用大语言模型（LLM）的对话能力协调多个机械臂完成协作任务。然而，LLM 存在严重的"幻觉"问题——它会提出违反物理约束的动作方案（如让机械臂去抓它够不到的物体），导致任务执行失败。

### 1.2 研究目标

设计一种强化学习（RL）方法，将 LLM 的"开放式问答"转化为"在合法动作集合中做选择题"，从而屏蔽幻觉、保证动作合法性，同时保留 LLM 的语义理解能力。

### 1.3 技术路线

```
场景观测 → 合法性 mask 生成 → RL 高层策略（MAPPO）→ 离散动作选择
                                    ↑
                            LLM 候选（可选）
                                    ↓
            动作字符串 → 现有 Parser → IK + RRT → 底层执行
```

## 2. 实验环境

### 2.1 硬件环境

- CPU: Intel Core（Windows 11 Pro）
- GPU: CUDA 12.1（PyTorch 2.5.1）
- 内存: 足够运行 MuJoCo 多臂仿真

### 2.2 软件环境

- Python 3.x + PyTorch 2.5.1
- MuJoCo（dm_control）物理仿真
- 智谱 GLM-4-flash API（LLM 基线）

### 2.3 任务环境

**Sort 任务**（主实验）：
- 3 个机械臂（Alice、Bob、Chad）
- 7 个面板（panel1-7）组成流水线
- 3 个方块需要放到各自的目标面板
- 每个机械臂只能够到相邻的 3 个面板
- 需要跨臂中转协作才能完成

**Sweep 任务**（跨任务验证）：
- 2 个机械臂（Alice 持簸箕、Bob 持扫帚）
- 3 个方块需要扫入簸箕再倒入垃圾桶
- 需要两臂同步协作（Alice 就位 → Bob 扫入）

## 3. 方法设计

### 3.1 动作空间

采用因子化离散动作编码：

```
动作 = (动词, 目标)
Sort:  动词 ∈ {WAIT, PICK·PLACE}，目标 ∈ {panel1, ..., panel7}
Sweep: 动词 ∈ {WAIT, MOVE, SWEEP, DUMP}，目标 ∈ {cubes, trash_bin, self}
```

两个 softmax head 分别输出动词和目标的概率，再用合法性 mask 屏蔽非法组合。

### 3.2 合法性 Mask

每一步根据当前状态动态生成布尔 mask，屏蔽以下非法动作：
- 物体不在该机械臂的可达范围内
- 手上已有物体时再次 PICK
- 目标面板不在可达范围内
- 违反任务规则的物体-面板搭配
- 已完成的方块不允许再被拾取

### 3.3 状态空间

每个智能体的观测向量包含：
- 物体位置编码（one-hot panel 编码，3×7=21 维）
- 各智能体持有状态（3 维）
- 任务进度标志（3 维）
- 智能体身份编码（3 维）

总计 30 维（Sort 任务）/ 19 维（Sweep 任务）。

### 3.4 奖励函数

| 奖励项 | 数值 | 触发条件 |
|--------|------|----------|
| 子目标完成 | +10 | 方块首次到达正确面板 |
| 全局成功 | +50 | 所有方块就位 |
| 势能塑形 | γΦ(s')−Φ(s) | 物体离目标更近 |
| 协作中转 | +2 | 把物体移入他人可达范围 |
| 非法动作 | −2 | mask 未屏蔽的非法操作 |
| 时间惩罚 | −0.05 | 每步固定 |

### 3.5 训练算法

MAPPO（Multi-Agent PPO）：
- 参数共享的 Actor（各智能体共用网络权重）
- 中心化 Critic（训练时看全局状态）
- MaskedCategorical 分布（采样时屏蔽非法动作）
- 分布式执行（每个智能体只看局部观测）

## 4. 实验结果

### 4.1 主对比实验（Sort 任务，符号化环境，200 局）

| 方法 | 成功率 | 平均步数 | 非法动作数 | 平均奖励 |
|------|-------:|--------:|-----------:|---------:|
| LLM-only (GLM-4-flash) | **0%** (0/3) | 8 (超时) | — | — |
| 随机策略（无 mask） | 1.0% | 29.9 | 80.2 | −162.2 |
| 随机策略（有 mask） | 65.5% | 22.1 | 7.5 | 54.2 |
| 启发式规则 | 22.0% | 24.3 | 0.0 | 25.9 |
| **MAPPO + mask（本文方法）** | **100%** | **3.6** | **0.3** | **89.1** |

**分析**：
- LLM-only 在 3 个不同初始布局上全部失败，典型错误是反复将 pink_polygon 放到 panel2（应为 panel4），即使环境反馈"非法"也无法自我纠正。
- 仅加入 mask（随机策略）就能将成功率从 1% 提升到 65.5%，说明约束本身极具价值。
- 启发式规则只能处理单步可达的情况，无法解决需要 Alice→Bob→Chad 三跳中转的场景。
- 本文方法在 3.6 步内 100% 完成任务，且几乎无非法动作。

![主对比实验](figures/fig_comparison.png)
*图 2：Sort 任务主对比实验——成功率（左）与平均步数（右）*

### 4.2 消融实验

| 配置 | 成功率 | 达到 80% 成功率所需步数 |
|------|-------:|----------------------:|
| 完整方法（mask + 塑形 + 协作奖励） | **100%** | 1,280 |
| 去掉 mask（训练时） | **1.5%** | 4,864 |
| 去掉势能塑形 | 100% | 2,048 |
| 去掉协作奖励 | 100% | 1,536 |

**关键发现**：去掉 mask 后成功率从 100% 暴跌至 1.5%。原因：训练时无约束，RL 学到了"作弊"策略（利用无约束环境的漏洞），上线后被合法性 mask 屏蔽，策略直接崩溃。这是本文核心假设"做选择题"的最强实证。

![消融实验](figures/fig_ablation.png)
*图 3：消融实验——去掉 mask 训练后成功率从 100% 暴跌至 1.5%*

### 4.3 LLM+RL 混合策略实验

| 提示策略 | 成功率 | 平均步数 | LLM 非法动作/步 |
|----------|-------:|--------:|----------------:|
| vanilla（LLM 只看任务描述） | 62.5% | 14.5 | 1.67 |
| **mask-aware（LLM 也看到合法动作集）** | **100%** | **4.25** | **0.35** |

**分析**：将合法性 mask 同时展示给 LLM，其幻觉率下降 4.7 倍（1.67→0.35 非法动作/步），混合策略成功率从 62.5% 提升至 100%。这证明约束信息是可跨范式共享的辅助信息——RL 用它训练，LLM 用它生成，二者共用一套 mask。

![Mask-Aware 消融](figures/fig_mask_aware.png)
*图 4：Mask-Aware LLM 消融——给 LLM 看合法动作集后幻觉率下降 4.7 倍*

### 4.4 跨任务泛化实验

| 任务 | 智能体数 | 动作空间 | 成功率 | 平均步数 | 训练时间 |
|------|:--------:|----------|-------:|--------:|---------:|
| Sort | 3 | PICK·PLACE × 7 panels | 100% | 3.6 | 24s |
| Sweep | 2 | MOVE/SWEEP/DUMP × 5 targets | 100% | 7.24 | 21s |

**关键论据**：切换任务时，训练代码（MAPPO 实现）**零修改**。只需要：
1. 填写新任务的动作词表（get_action_vocab）
2. 实现新任务的合法性规则（get_action_mask）
3. 编写对应的符号化训练环境

Sweep 任务学到的策略还自发发现了"流水线"优化：Alice 在倒垃圾时 Bob 已经在去下一个方块的路上，比顺序执行节省 2 步。

![跨任务泛化](figures/fig_cross_task.png)
*图 5：跨任务泛化——同一套训练代码在 Sort 和 Sweep 上均达到 100% 成功率*

### 4.5 真实 MuJoCo 物理仿真实验（Sort 任务）

| 配置 | 成功率 | 平均步数 | 备注 |
|------|-------:|--------:|------|
| 默认 | 40% (2/5) | 6.8 | 部分 seed 因物理漂移失败 |
| 加放置修正 | **60% (3/5)** | **5.6** | snap-finished 工程修复 |

**成功案例轨迹**（seed=1）：
```
step 0: Alice=PICK pink_polygon PLACE panel3
        Bob=PICK yellow_trapezoid PLACE panel5
        Chad=PICK blue_square PLACE panel5       ← 三臂并发中转
step 1: Bob=PICK blue_square PLACE panel3
        Chad=PICK yellow_trapezoid PLACE panel6  ← 接力
step 2: Alice=PICK blue_square PLACE panel2
        Bob=PICK pink_polygon PLACE panel4       ← 最终落位
→ 任务完成，3 步
```

**失败原因分析**：
- 2 个失败 seed 的原因是底层 RRT 多臂避障求解器在某些关节位姿下找不到无碰撞路径
- 这是 IK/RRT 层面的工程问题，不是 RL 策略问题
- 策略本身的动作选择始终正确（从符号化环境 100% 成功率可以确认）

### 4.6 LLM 失败模式对比

**Sort 任务 LLM 典型失败**：
```
Alice: PICK pink_polygon PLACE panel2
[环境反馈]: 失败，pink_polygon 只能放到 panel4/panel3/panel5，不能放 panel2
Alice: PICK pink_polygon PLACE panel2    ← 重复同样的错误
[环境反馈]: 失败...
（循环 7 轮后超时）
```

**Sweep 任务 LLM 典型失败**：
- LLM 能正确生成 MOVE/SWEEP 格式的动作
- 但无法协调 Alice 和 Bob 的时序（Bob 在 Alice 还没就位时就 SWEEP）
- 8 步内未能将任何方块送入垃圾桶

两个任务的共同模式：**LLM 缺乏对物理约束的内化理解**，即使收到环境反馈也无法有效自我纠正。

## 6. 结论

### 6.1 主要贡献

1. **提出"合法性 mask + RL 选择"的框架**，将 LLM 的开放式输出约束为合法动作集合中的选择问题，在 Sort 任务上实现 100% 成功率（vs LLM-only 0%）。

2. **通过消融实验证明 mask 是核心方法**，去掉 mask 训练后成功率从 100% 暴跌至 1.5%，直接验证了"做选择题"假设的必要性。

3. **发现约束信息可跨范式共享**，将同一套 mask 喂给 LLM 后其幻觉率下降 4.7 倍，混合策略成功率从 62.5% 提升至 100%。

4. **验证框架的跨任务通用性**，在 Sort 和 Sweep 两个不同任务上均达到 100% 成功率，且训练代码零修改。

### 6.2 局限性

1. 真实 MuJoCo 环境成功率（60%）低于符号化环境（100%），差距来自底层 IK/RRT 求解器的局限性，非策略问题。
2. 当前 API key 仅支持 GLM-4-flash（较弱模型），更强模型（GLM-4-plus）可能改善 LLM-only 基线表现。
3. 符号化训练环境不建模手臂间碰撞，导致策略倾向于并发动作，需要 runner 层面的顺序回退机制。

### 6.3 未来工作

1. 底层 IK/RRT 调优，提升真实环境成功率至 80%+
2. 接入更强 LLM（GPT-4、GLM-4-plus）重做混合策略实验
3. 将 mask-aware 提示工程扩展为"LLM 在线学习约束"的研究方向
4. 扩展到更多 RoCo 任务（cabinet、pack、rope）

## 7. 复现指南

### 7.1 环境配置

```bash
# 依赖：PyTorch, dm_control, mujoco, numpy, matplotlib
pip install torch dm_control mujoco numpy matplotlib
```

### 7.2 一键复现

```powershell
# 训练（CPU 即可，各 ~30 秒）
python train_rl_sort.py --steps 30000 --save checkpoints/sort_mappo.pt
python train_rl_sweep.py --steps 30000 --save checkpoints/sweep_mappo.pt

# 对比实验（~2 分钟）
python benchmark_rl.py --train-steps 25000 --eval-episodes 200

# 消融实验 + 训练曲线（~3 分钟）
python make_training_curves.py --steps 25000

# Mask-aware LLM 消融（需要 API，~3 分钟）
python benchmark_mask_aware.py --episodes 8 --llm-model glm-4-flash

# 真实 MuJoCo 评估（~5 分钟）
python run_rl_sort.py --method mappo --load checkpoints/sort_mappo.pt \
    --num-runs 5 --max-steps 8 --snap-finished

# 生成答辩图
python make_plots.py
```

### 7.3 文件结构

```
rocobench/rl/                    - RL 模块（8 个文件）
train_rl_sort.py / sweep.py      - 训练入口
run_rl_sort.py / sweep.py        - 真实环境评估
benchmark_*.py                   - 对比/消融脚本
make_plots.py / curves.py        - 可视化
figures/                         - 5 张答辩图 + 训练曲线数据
data/                            - 实验日志 + 视频
checkpoints/                     - 模型权重
```
